Android 代码规范

注释

Java 程序有两类注释：
实现注释(implementation comments)和文档注释(document comments)。
实现注释是使用/*…*/和//界定的注释。
文档注释(被称为”doc comments”)由/**…*/界定。文档注释可以通过javadoc 工具转换成HTML 文件。

1.类注释

每一个类都要包含文档注释,以说明当前类的功能
　　a)类注释模板设置步骤:
　　　Android Studio -> File -> Settings -> 左上角搜索 File and Code Templates -> 右边面板选择Class -> 复制如下模板 -> 保存(Apply)

注意：这里所设置的注释模板只有在创建新的类的时候才会自动生成出现。若想在自己的代码中快速设置注释还需要使用第二种设置方式
　　b)自定义模板设置步骤(重要):
　　　Android Studio -> File -> Settings -> 左上角搜索 Live Templates -> 右边面板选择Add(右侧”+”号) -> 选择Template Group创建模板组-> 输入组名 custom comment -> 点击新建的组 ->点击”+”..Live Template-> 复制如下模板到Template text编辑框内

-> 当模板配置如下图保存(Apply)

custom comment组中还可定义field comment(fc)按照类自定义注释模板自行添加,原理类似.
配置完后在类或成员变量的上方输入自定义快捷键,回车即可使用.

另:不建议在一个源文件中定义多个类,尽量把类分布到对应目录结构中,依赖性极强的类除外.
例如:JobInfo.Builder->/** Builder class for constructing {@link JobInfo} objects. */

2.方法注释

不要求所有方法都添加注释,但容易产生歧义或功能不清的方法需添加文档注释。
Tips:注释有一种快捷操作供参考:
　　Android Studio -> File -> Settings -> 左上角搜索Fix doc comment->双击后选择add keyboard shortcut ->自定义快捷键(建议Alt+Shift+K)->OK
将光标放到方法声明或实现内(成员变量放在变量名上)输入自定义快捷键即可.

3.类成员变量和常量注释

类成员变量和常量需要使用文档注释,通常情况注释写在声明的上方

4.其他注释

方法内部的注释 如果需要多行 使用/*…… */形式，如果为单行是用//……形式的注释。不要再方法内部使用 java doc 形式的注释“/**……**/”.
方法内部注释为了不影响上下文间的逻辑连贯性,可以全在行尾添加注释,也可全在行上方添加注释,但不要两个混着使用而降低代码可读性.
